CVAULT-167: Fix TradingPerformance Table Horizontal Scroll on Tablet
Date: 2026-02-09
Status: COMPLETED
================================================================================

ISSUE DESCRIPTION:
The TradingPerformance table component had horizontal scroll overflow issues on 
tablet breakpoints (768px-1024px). The table used a fixed min-w-[640px] which 
forced horizontal scrolling on smaller tablet screens, making it difficult for 
users to view all trading data.

ROOT CAUSE:
1. Fixed minimum width of 640px (min-w-[640px]) on table
2. All 8 columns visible on all screen sizes ≥ 640px
3. No responsive column hiding for narrower tablet viewports

SOLUTION IMPLEMENTED:

File: src/components/TradingPerformance.tsx

1. Responsive Table Width (Line 217):
   BEFORE: <table className="w-full text-sm min-w-[640px]">
   AFTER:  <table className="w-full text-sm min-w-[500px] lg:min-w-[640px]">
   
   EFFECT:
   - Small tablets (640px-1023px): 500px minimum width
   - Desktop (1024px+): 640px minimum width
   - Reduces horizontal scroll on tablets by 140px

2. Responsive Column Hiding:
   
   a) P&L Column (Lines 225, 246):
      BEFORE: Always visible on sm+ (640px+)
      AFTER:  hidden md:table-cell (visible on 768px+)
      
      BREAKPOINTS:
      - sm (640px-767px): HIDDEN
      - md+ (768px+): VISIBLE
      
   b) Consensus Column (Lines 226, 258):
      BEFORE: Always visible on sm+ (640px+)
      AFTER:  hidden lg:table-cell (visible on 1024px+)
      
      BREAKPOINTS:
      - sm-md (640px-1023px): HIDDEN
      - lg+ (1024px+): VISIBLE
